How to add addons

# Overview

To create your server addons, you must use the **ContentMaker** utility. This tool simplifies the process of indexing and packaging your content for the server. If you want to edit player model attaches or get model anim frames, you can take models.exe from CB2 Dedicated [Server](class_server.md) in Tools\ContentMaker

# Packing Modes

When you launch the application, you will be presented with three packing modes:

![](examples/exa.png)

ContentMaker Packing Modes

- **Mode 1: Scan & Index** Scans the specified folder and generates an `addons.jsonc` file. Use this if you host files individually on your own web server.
- **Mode 2: Pack to .cbpak** Scans the folder, generates `addons.jsonc`, and bundles all files into a single `.cbpak` container. Ideal for cloud storage like Google Drive.
- **Mode 3: Unpack** Allows you to extract files from an existing `.cbpak` archive.

---

# Configuration

After scanning or packing, you need to configure the generated `addons.jsonc` file.

![](examples/exa2.png)

Example of addons.jsonc structure

1. **Edit JSON:** Open `addons.jsonc` and fill in the required metadata.
2. **Upload:** Upload the files (and `.cbpak` if used) to your hosting.
3. **Link:** Copy the direct download link and paste it into the `contenturl` setting in your server configuration.

Note
:   Make sure your physical files are located in the server's `addons/` folder, and their paths in `addons.jsonc` match this structure.

See also
:   [View addons.jsonc example](examples/addons.jsonc)
:   [View addons.jsonc with original objects](examples/originals.jsonc)

# Adding New Rooms

To add new rooms to your addon:

1. Create a new `.ini` file (similar to `Data/rooms.ini`).
2. Add only your new room definitions.
3. You can override existing rooms by name, but the `roomid` will be inherited from the original.

![](examples/exa3.png)

New room templates configuration

Important
:   You must specify the path to this file in the `roomtemplates` field within your `addons.jsonc`.

# Important Rules & Limits

Warning
:   Failure to follow these rules may result in client-side errors or failed downloads.

- **Root Pathing:** Always package files from the **content root**. Do not include the "server" folder itself, or paths will be duplicated (e.g., `addons/addons/file.txt`).
- **File Size:** Individual files cannot exceed **256 MB**.
- **Archive Size:** A single `.cbpak` file is limited to **1.5 GB**. If exceeded, the tool will automatically split it into multiple parts.
- **Security:** Executable files (`.exe`, `.dll`, etc.) are **strictly prohibited** and will not be packed.
- **File hash:** If you enter an incorrect hash of the downloaded file, this file will be downloaded every time when you connect to the server.
